# UI Validation Checklist

Run this checklist before finishing.

## Functional checks

- All required states exist (loading, empty, error, success where applicable)
- Primary flows are operable with keyboard and pointer
- Form controls have labels and clear error states

## Visual checks

- Direction and differentiator are visible in final result
- Typography hierarchy is clear and consistent
- Spacing rhythm is consistent across sections and breakpoints
- Colors support contrast and readable hierarchy

## Responsiveness checks

- Layout works on mobile and desktop targets
- No clipped text or overlapping controls
- Hit targets remain usable on touch devices

## Quality gates

- Run applicable lint/test/build commands
- Call out any checks not run and why
- List concrete changed files in delivery message
