# Product Portfolio

## Core Principle

Design the portfolio for hiring decisions, not self-expression first. Start with the role and story the candidate needs a hiring manager to understand in the first few minutes, then choose the work, structure, writing, and visuals that prove it.

## Workflow

1. Define the target.
   - Identify the role level, role type, company type, and 3 skills that must stand out.
   - Write the one-sentence takeaway a reviewer should remember after 3 minutes.
   - Do not start with layout until this target is clear.

2. Select evidence.
   - Prefer 2-4 sharp case studies over a long gallery.
   - Order cases by relevance to the target role, strength of outcome, and clarity of story.
   - For student or side projects, frame them like real projects: real problem, process, feedback, tests, and outcome.
   - For NDA work, anonymize company names, exact numbers, and screens if needed; preserve the decision process, constraints, role, tradeoffs, and measurable direction of impact.

3. Structure the portfolio.
   - Homepage: state role positioning, strongest proof, selected work, credibility signals, and a clear contact path.
   - Case studies: make each case scannable before it becomes detailed.
   - About: explain background only when it strengthens hiring fit.
   - Contact: make outreach obvious and low-friction.

4. Tell each case study with C2P-R.
   - Context: product, audience, team, timeline, candidate role, constraints.
   - Challenge: business/user problem, stakes, why it mattered.
   - Process: decisions, tradeoffs, research, iterations, collaboration, validation.
   - Result: outcome, learning, impact, what changed, and what the candidate would do next.

5. Choose format and tools.
   - Use the simplest format that lets hiring reviewers scan fast and inspect depth.
   - Website portfolios should prioritize speed, navigation clarity, responsive behavior, and readable case studies.
   - Deck/PDF portfolios should work as interview support, not as the only public artifact unless the user explicitly needs that.

6. Use AI carefully.
   - Use AI to outline, compress, clarify, generate critique prompts, and create alternate versions for role targets.
   - Do not let AI invent metrics, claims, research, employers, or responsibilities.
   - Rewrite AI output so it sounds specific to the designer and the project.

## Output Standards

- Lead with direct recommendations and concrete rewritten copy, not generic advice.
- Ask for missing target-role details only when they materially change portfolio decisions; otherwise make a reasonable assumption and label it.
- Keep portfolio copy skimmable: short headings, short paragraphs, proof-first bullets, and clear project metadata.
- Separate strategy notes from candidate-facing copy so the user can paste usable content without cleanup.
- Flag unverifiable claims, missing outcomes, weak project selection, unclear role contribution, and buried contact paths.
